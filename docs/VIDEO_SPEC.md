## 1. 영상 포맷 스펙 (Video Format Specification)

### 1.1 기본 포맷

| 항목 | 값 |
|-----|---|
| 해상도 | 1080 × 1920 px (9:16 세로) |
| 프레임레이트 | 30 fps |
| 비디오 코덱 | H.264 (libx264) |
| 오디오 코덱 | AAC (aac) |
| 오디오 샘플레이트 | 44,100 Hz |
| 오디오 채널 | Stereo (2ch) |
| 컨테이너 | MP4 (.mp4) |
| 색상 공간 | YUV 4:2:0 (yuv420p) |
| 최대 비트레이트 (비디오) | 8 Mbps |
| 최대 비트레이트 (오디오) | 192 kbps |
| 최대 용량 | 100 MB |
| 최대 재생 시간 | 60초 |

### 1.2 인코딩 프리셋

```
ffmpeg -vf scale=1080:1920 -c:v libx264 -preset slow -crf 23
       -c:a aac -b:a 192k -ar 44100 -ac 2
       -pix_fmt yuv420p -movflags +faststart
       -maxrate 8M -bufsize 16M
```

- `-preset slow`: 파일 크기 최적화 우선 (빠른 처리가 필요하면 `medium` 사용)
- `-crf 23`: 시각적 품질 기준값 (낮을수록 고품질, 18–28 범위)
- `-movflags +faststart`: 스트리밍을 위한 moov atom 앞 배치

## 2. 영상 구성 섹션 (Video Composition Sections)

모든 영상은 아래 3개 섹션으로 구성된다.

```
┌─────────────────────────────┐
│  인트로 (Intro)   — 2초     │
│  버킷리스트 아이템 제목 표시 │
├─────────────────────────────┤
│  메인 콘텐츠 (Main Content) │
│  사용자 업로드 미디어 재생  │
│  (최소 3초 ~ 최대 56초)     │
├─────────────────────────────┤
│  아웃트로 (Outro)  — 2초    │
│  앱 브랜딩 + 완료 메시지    │
└─────────────────────────────┘
```

### 2.1 인트로 (Intro) — 2초 고정

- 배경: 템플릿 배경색 또는 그라디언트
- 중앙 텍스트: 버킷리스트 아이템 제목 (애니메이션 fade-in)
- 좌상단: 앱 로고 워터마크 (소형)
- 트랜지션 아웃: 다음 섹션으로 부드러운 dissolve

### 2.2 메인 콘텐츠 (Main Content) — 가변 (3초 ~ 56초)

- 사용자가 업로드한 사진/영상 클립을 순차 재생
- 각 미디어 아이템 표시 시간:
  - 사진 (image): 기본 3초, 최대 5초
  - 영상 (video): 원본 길이 그대로 (최대 15초/클립)
- 텍스트 오버레이: 사용자 메모에서 추출한 캡션 (하단)
- 미디어 간 트랜지션: 템플릿에 따라 다름

### 2.3 아웃트로 (Outro) — 2초 고정

- 배경: 인트로와 동일한 배경 스타일
- 중앙 텍스트: "완료! ✓" 또는 커스텀 완료 메시지
- 하단: 앱 브랜딩 ("Bucket List App으로 제작")
- 트랜지션 인: 메인 콘텐츠에서 fade-in

## 3. 템플릿 타입 (Template Types)

총 3가지 템플릿을 제공한다.

| 템플릿 | 분위기 | 대상 콘텐츠 |
|-------|-------|-----------|
| `minimal` | 깔끔하고 모던한 미니멀리즘 | 음식, 문화, 일상 |
| `vibrant` | 활기차고 컬러풀한 에너지 | 여행, 피트니스, 야외 활동 |
| `film` | 시네마틱한 영화적 감성 | 여행, 문화, 특별한 순간 |

## 4. 각 템플릿 레이아웃 상세 (Template Layout Details)

### 4.1 Minimal 템플릿

**배경:**
- 기본 배경색: `#FFFFFF` (흰색) 또는 `#F5F5F5` (연회색)
- 인트로/아웃트로 오버레이: 없음

**텍스트:**
- 제목 폰트: Noto Sans KR Bold, 48px
- 제목 색상: `#1A1A1A`
- 제목 위치: 화면 중앙 (x: 50%, y: 45%)
- 캡션 폰트: Noto Sans KR Regular, 24px
- 캡션 색상: `#333333`
- 캡션 위치: 하단 (x: 50%, y: 88%)
- 캡션 배경: 반투명 흰색 패널 (`rgba(255,255,255,0.85)`)

**미디어 클립 영역:**
- 미디어 배치: 전체 화면 (1080×1920), object-fit: cover
- 상하단 여백 없음 (full bleed)

**트랜지션:**
- 타입: `fade`
- 지속 시간: 0.5초

**오버레이:**
- 상단 그라디언트: `rgba(0,0,0,0)` → `rgba(0,0,0,0.3)` (상단 20% 영역)
- 하단 그라디언트: `rgba(0,0,0,0)` → `rgba(0,0,0,0.4)` (하단 30% 영역)
- 캡션 영역 배경: 별도 반투명 패널

---

### 4.2 Vibrant 템플릿

**배경:**
- 인트로/아웃트로 그라디언트: `#FF6B35` → `#F7C59F` (세로 방향, 좌상 → 우하)
- 대체 그라디언트 세트: `#667EEA` → `#764BA2`

**텍스트:**
- 제목 폰트: Noto Sans KR ExtraBold, 52px
- 제목 색상: `#FFFFFF`
- 제목 위치: 화면 상단 1/3 (x: 50%, y: 30%)
- 제목 배경: 없음 (drop shadow만 적용)
- 제목 효과: `text-shadow: 2px 2px 8px rgba(0,0,0,0.5)`
- 캡션 폰트: Noto Sans KR Bold, 26px
- 캡션 색상: `#FFFFFF`
- 캡션 위치: 하단 (x: 50%, y: 85%)
- 캡션 배경: 반투명 컬러 패널 (브랜드 색상 `rgba(255,107,53,0.75)`)

**미디어 클립 영역:**
- 미디어 배치: 중앙 집중형 (1080×1350), 상단 570px은 인트로 그라디언트와 블렌딩
- 또는 전체 화면 커버

**트랜지션:**
- 타입: `slide`
- 방향: 좌→우 (slide_left)
- 지속 시간: 0.4초

**오버레이:**
- 전체 화면 그라디언트 오버레이: `rgba(255,107,53,0.15)` (브랜드 색상 틴트)
- 하단 패널: 높이 180px, `rgba(0,0,0,0.55)`

---

### 4.3 Film 템플릿

**배경:**
- 인트로/아웃트로 배경: `#0D0D0D` (거의 검정)
- 필름 그레인 텍스처 오버레이 적용

**텍스트:**
- 제목 폰트: Noto Serif KR, 44px (Italic)
- 제목 색상: `#F0E6D3` (따뜻한 크림색)
- 제목 위치: 화면 하단 1/3 (x: 50%, y: 72%)
- 제목 하단 장식선: 2px 가로 구분선 (`#C5A55A`, 황금색)
- 캡션 폰트: Noto Serif KR Light, 22px
- 캡션 색상: `#D4C5B0`
- 캡션 위치: 하단 (x: 50%, y: 87%)

**미디어 클립 영역:**
- 미디어 배치: 전체 화면 커버 (1080×1920)
- 시네마틱 레터박스: 상하 60px 검정 바 (선택적)

**트랜지션:**
- 타입: `zoom`
- 방향: 확대 (zoom_in, 1.0 → 1.05)
- 지속 시간: 0.8초

**오버레이:**
- 전체 화면: `rgba(0,0,0,0.25)` 어두운 틴트
- 필름 그레인: noise 텍스처 오버레이 (opacity: 0.08)
- 비네팅 효과: 외곽 어둡게 처리 (radial gradient)
- 색상 그레이딩: 약간 탈채도 (saturation: 0.85) + 따뜻한 색온도

## 5. 지원 미디어 입력 타입 (Supported Media Input Types)

### 5.1 입력 미디어 조합

| 모드 | 설명 | 최소 | 최대 |
|-----|-----|-----|-----|
| 사진만 (Photos Only) | JPEG/PNG/WEBP 이미지만 사용 | 1장 | 20장 |
| 영상만 (Videos Only) | MP4/MOV/AVI 클립만 사용 | 1개 | 5개 |
| 혼합 (Mixed) | 사진과 영상 클립 혼합 | 1개 | 사진 20장 + 영상 5개 |

### 5.2 입력 미디어 제약 조건

**이미지:**

| 항목 | 조건 |
|-----|-----|
| 지원 포맷 | JPEG, PNG, WEBP, HEIC |
| 최소 해상도 | 480 × 480 px |
| 최대 파일 크기 | 10 MB |
| 색상 모드 | RGB (CMYK 불가) |

**영상:**

| 항목 | 조건 |
|-----|-----|
| 지원 포맷 | MP4 (H.264), MOV (H.264/H.265), AVI |
| 최소 해상도 | 480 × 480 px |
| 최대 파일 크기 | 200 MB |
| 최대 클립 길이 | 60초 (초과분은 앞에서 자름) |
| 최소 클립 길이 | 1초 |

### 5.3 처리 파이프라인 (입력 → 정규화 → 합성)

```
입력 미디어
    │
    ▼
[입력 검증]
 - 포맷 체크
 - 크기/길이 제한
 - 최소 품질 확인
    │
    ▼
[정규화]
 - 해상도 → 1080×1920 리스케일 (object-fit: cover)
 - 사진 → 3초 정지 영상 변환 (with Ken Burns 효과 선택적)
 - 영상 → 최대 길이 트리밍
 - 오디오 스트립 (BGM 별도 처리)
    │
    ▼
[합성]
 - 인트로 + 메인 클립들 + 아웃트로 순서 연결
 - 텍스트/오버레이 합성
 - BGM 믹싱
    │
    ▼
[인코딩 & 출력]
 - H.264 인코딩
 - MP4 컨테이너 패키징
 - 썸네일 추출 (첫 번째 클립 1초 시점)
```

## 6. BGM 처리 방식 (BGM Processing)

### 6.1 볼륨 정규화 (Volume Normalization)

- 목표 라우드니스: **-16 LUFS** (Instagram Reels / YouTube Shorts 권장값)
- 피크 제한: **-1 dBTP** (트루 피크)
- 측정 도구: `ffmpeg-normalize` 또는 ffmpeg `loudnorm` 필터

```
ffmpeg -i input.mp3 -af loudnorm=I=-16:TP=-1:LRA=11:measured_I=-14:
       measured_LRA=7:measured_TP=-2:measured_thresh=-24:offset=0
       -ar 44100 output_normalized.aac
```

### 6.2 페이드 인/아웃 (Fade In/Out)

| 구간 | 처리 |
|-----|-----|
| 인트로 시작 (0s ~ 1.5s) | 페이드 인 (0 → 100% 볼륨) |
| 아웃트로 종료 (마지막 2s) | 페이드 아웃 (100% → 0% 볼륨) |

```python
# moviepy audio fade example
bgm_clip = bgm_clip.audio_fadeout(2.0).audio_fadein(1.5)
```

### 6.3 루프 처리 (Loop Handling)

- 영상 총 길이 > BGM 길이인 경우: BGM을 루프 반복 후 크로스페이드(0.5초)로 이음새 처리
- 영상 총 길이 < BGM 길이인 경우: BGM을 영상 길이에 맞게 잘라냄

### 6.4 볼륨 레벨

- BGM 기본 볼륨: **70%** (사용자 목소리가 포함된 영상 클립과의 균형 고려)
- 원본 영상 오디오 제거 후 BGM만 사용 (별도 설정 없는 경우)

### 6.5 지원 BGM 포맷

| 포맷 | MIME 타입 |
|-----|---------|
| MP3 | audio/mpeg |
| AAC | audio/aac |
| M4A | audio/mp4 |
| WAV | audio/wav |

## 7. 썸네일 생성 (Thumbnail Generation)

- 추출 시점: 첫 번째 메인 클립의 1초 시점 프레임
- 해상도: 540 × 960 px (원본의 50% 다운스케일)
- 포맷: JPEG (quality: 85)
- 저장 경로: `videos/{user_id}/{video_id}_thumb.jpg`

## 9. 오류 처리 (Error Handling)

| 오류 코드 | 설명 | 처리 방법 |
|---------|-----|---------|
| `MEDIA_INVALID_FORMAT` | 지원하지 않는 미디어 포맷 | 해당 파일 건너뜀, 나머지 처리 |
| `MEDIA_TOO_LARGE` | 파일 크기 초과 | 거부, 사용자에게 에러 반환 |
| `MEDIA_TOO_SHORT` | 최소 길이 미달 (영상 < 1초) | 건너뜀 |
| `TEMPLATE_NOT_FOUND` | 존재하지 않는 템플릿 | `minimal` 폴백 사용 |
| `BGM_LOAD_FAILED` | BGM 파일 로드 실패 | BGM 없이 처리 계속 |
| `FFMPEG_ERROR` | FFmpeg 인코딩 오류 | 태스크 실패, `generated_videos.status = 'failed'` |
| `OUTPUT_TOO_LARGE` | 결과물이 100 MB 초과 | CRF 값을 높여 재인코딩 |
