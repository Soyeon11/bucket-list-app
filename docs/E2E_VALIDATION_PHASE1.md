# Phase 1 E2E 검증 체크리스트

> 작성일: 2026-05-09  
> 검증 기준: 실제 기기(iOS Expo Go) + FastAPI 서버 로컬 실행 환경  
> 기록 방법: 각 항목 결과에 ✅ (통과) / ❌ (실패) / ⏭️ (스킵) 기입

---

## 사전 준비

- [ ] FastAPI 서버 실행 확인: `http://localhost:8000/health` → `{"status": "ok"}`
- [ ] Expo Metro 번들러 실행 확인: `http://localhost:8081`
- [ ] iOS Expo Go 앱 실행 후 QR 코드 스캔
- [ ] Supabase에 테스트 계정 1개 준비 (이메일/비밀번호)

## UC-01. 인증 (Authentication)

### UC-01-1. 이메일/비밀번호 로그인 성공
| 항목 | 내용 |
|------|------|
| **전제조건** | Supabase에 가입된 이메일 계정 존재 |
| **테스트 절차** | 1. 앱 실행 → 로그인 화면 확인<br>2. 이메일, 비밀번호 입력<br>3. 로그인 버튼 탭 |
| **기대 결과** | 홈 탭(`/`) 화면으로 이동, 상단에 닉네임 표시 |
| **결과** | ❌ 홈 탭 이동은 정상이나 닉네임 미표시 ("사용자님" fallback 출력) |
| **원인** | `app/_layout.tsx`에서 세션 획득 후 `profiles` 테이블 fetch 누락 → `profile` 항상 null |
| **픽스** | `app/_layout.tsx`에 `fetchProfile()` 추가, 세션 설정 시 호출하도록 수정 ✅ |
| **재검증 결과** | 홈 탭 화면으로 이동, 상단에 닉네임 표시 |

### UC-01-2. 이메일/비밀번호 로그인 실패
| 항목 | 내용 |
|------|------|
| **전제조건** | 없음 (로그아웃 상태) |
| **테스트 절차** | 1. 로그인 화면에서 잘못된 비밀번호 입력<br>2. 로그인 버튼 탭 |
| **기대 결과** | 에러 알림 표시, 화면 이동 없음 |
| **결과** | ⏭️ UC-01-1 검증 후 로그아웃 수단 없어 검증 불가 |
| **원인** | 앱에 로그아웃 버튼 미구현 |
| **픽스** | 홈 화면 우상단에 "로그아웃" 버튼 추가, 확인 알림 포함 ✅ |
